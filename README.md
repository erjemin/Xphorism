###### Исходное задание
Есть некая тестовая база, регулярно обновляемая извне. Она содержит 2 таблицы: Contacts и Telephones. В таблице Contacts хранятся ФИО людей и адреса в Telephones – телефонные номера. Связь с таблицей Contacts осуществляется при помощи внешнего ключа. СУБД можно выбрать любую.

Клиентская часть – SPA на HTML (css,js,jq…), страница содержит таблицу представления контактов (ФИО – телефон – адрес). Через заданный интервал времени в секундах таблица автоматически подгружает новые строки из БД.

Для выборки данных из таблицы применяется web-API, очень желательно применить Django REST Framework. При загрузке номеров телефонов нужно применять маску «+9(999)999-99-99». В случае наличия для одного контакта нескольких номеров необходимо сгруппировать их по ФИО данного контакта.

Как результат приложить сам проект и краткую инструкцию по запуску/развертыванию.
